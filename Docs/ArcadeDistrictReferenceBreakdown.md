# Arcade District Reference Breakdown

目标：把用户提供的赛博街机俯视参考图拆成可持续迭代的 Unity 分层场景，而不是整图贴背景。

坐标说明：
- 使用归一化参考坐标，左下角为 `(0, 0)`，右上角为 `(1, 1)`。
- 后续拿到原始图片文件后，再按像素坐标做精确切层。
- Unity 房间建议映射为宽 `16`、高 `9` 的单屏俯视房间。

## 视觉方向

- 主题：霓虹街机区 / Arcade District。
- 视角：俯视 2.5D，略带等距感。
- 主色：深蓝紫环境光，洋红/青色霓虹强调。
- 场景结构：中心战斗空地，四周墙体和门，边缘装饰密集，中间保留清晰移动空间。
- 游戏性要求：所有可交互/可碰撞对象必须从背景拆出，单独绑定现有脚本。

## 背景层

这些内容合成一张或多张背景图，不绑定交互逻辑。

### BG_00_Floor

- 内容：整块蓝紫色方砖地面、湿润反光、地砖裂纹和轻微污渍。
- 范围：`x 0.08-0.92`，`y 0.10-0.86`。
- Unity：`SpriteRenderer`，sortingOrder `-30`。
- 碰撞：无。

### BG_01_Walls

- 内容：四周厚墙、墙顶、墙角、边缘暗部。
- 范围：全图外围。
- Unity：可做成四段墙背景图，也可以拆为 North/South/East/West。
- 碰撞：不直接挂在背景层，碰撞用独立 Collision Layer。

### BG_02_Lighting

- 内容：紫色/青色地面反光、门口投光、设备发光晕染。
- 范围：全图。
- Unity：建议单独半透明叠加层，sortingOrder `20`，不参与碰撞。
- 注意：如果性能优先，可与地面背景合并。

### BG_03_Decor_NoLogic

- 内容：远处植物、墙外暗部、边缘无法交互的空调/风扇、远景广告残影。
- 范围：主要在房间边缘。
- Unity：可拆成数个装饰 Sprite，sortingOrder `-10` 到 `5`。
- 碰撞：无或只用墙体碰撞替代。

## UI 参考，不进入场景层

参考图里有 HUD 和小地图，但这次拆场景时不把 UI 画进背景：

- 左上：HP、能量、技能图标。
- 右上：时间、区域名、威胁等级、货币。
- 右侧：小地图、目标。
- 左下：Combo 和伤害倍率。

当前项目 UI 后续应单独做 `UI Reference`，不要和场景参考混在一张图里。

## 分层合入策略

### 阶段 1：Reference Overlay

- 保存原始参考图到 `Assets/Art/References/ArcadeDistrict/reference.png`。
- 在 Unity 场景里生成 `Reference Overlay`，半透明显示。
- 目的：校准坐标、比例、构图。

### 阶段 2：Playable Greybox

- 用纯色/现有测试 Sprite 生成可玩场景。
- 所有组件按本表坐标摆放。
- 碰撞、交互、传送、战斗全部先跑通。

### 阶段 3：Layered Art

- 背景层：地面、墙、灯光、远景装饰。
- 组件层：门、宝箱、NPC、终端、街机、能量柱、敌人、Boss。
- 碰撞层：独立 Collider，不依赖图片边缘。

### 阶段 4：1:1 校准

- 打开 `Reference Overlay`。
- 用 Scene View 对齐：
  - 背景外轮廓。
  - 四门中心点。
  - 中央空场范围。
  - 每个功能组件位置和大小。
- 关闭 Overlay 后必须仍然是分层可玩场景。
